# Topic Cluster Playbook — Schritt für Schritt

> Ein Cluster vollständig ausbauen, bevor der nächste beginnt.
> Topical Authority entsteht durch Tiefe, nicht durch viele halbfertige Cluster.

---

## Warum Topic Cluster?

Websites mit klarer Cluster-Architektur erzielen 34% mehr organischen Traffic-Wachstum
als Sites mit einzelnen Keyword-Seiten (HireGrowth Analysis 2025).

Google bewertet nicht mehr einzelne Seiten isoliert — es bewertet die thematische Tiefe
einer Domain zu einem Thema. Wer alles über KI-Automatisierung publiziert und
alles intern verlinkt, rankt für "KI Automatisierung" besser als eine einzelne perfekte Seite.

---

## Anatomie eines Clusters

```
PILLAR PAGE (Haupt-URL: /thema/)
│
├── Cluster Page 1: /thema/subtopic-a/
├── Cluster Page 2: /thema/subtopic-b/
├── Cluster Page 3: /thema/subtopic-c/
├── Cluster Page 4: /thema/subtopic-d/
├── Cluster Page 5: /thema/subtopic-e/
└── ... (6–12 Cluster Pages optimal)
```

**Pillar Page:** Breites Thema, 1.500–3.000 Wörter. Gibt Überblick.
Verlinkt auf alle Cluster Pages.

**Cluster Page:** Ein spezifisches Subtopic, 800–1.500 Wörter. Geht in die Tiefe.
Verlinkt zurück zur Pillar Page + optional zu 1–2 anderen Cluster Pages.

---

## Schritt-für-Schritt: Einen Cluster aufbauen

### Schritt 1: Cluster-Thema wählen

Das Cluster-Thema muss:
- Ein echtes Geschäftsfeld des Kunden abdecken
- Genug Subtopics haben (mindestens 6 sinnvolle Cluster Pages)
- Transaktionalen oder Problem-Awareness-Intent haben (nicht rein informationell)

Faustregel: Wenn du für das Thema mehr als 8 verschiedene Fragen findest die
potenzielle Kunden googeln → gutes Cluster-Thema.

### Schritt 2: Keyword-Set aufbauen

Finde für das Cluster-Thema:
- 1 Pillar Keyword (breit, 500–5.000 Suchen/Monat)
- 6–12 Cluster Keywords (spezifisch, 50–500 Suchen/Monat)
- Semantische Variationen für jeden Artikel (2–4 pro Seite)

Quellen: Google Keyword Planner (kostenlos), Google Autosuggest,
"People Also Ask" Box, Answer The Public

### Schritt 3: Pillar Page schreiben

Die Pillar Page beantwortet die Frage: "Alles über [Thema] für [Zielgruppe]"

Struktur:
```
H1: [Thema] — [Nutzen für Zielgruppe]
Intro: 200 Wörter, beantwortet die Kernfrage sofort (GEO-Pflicht)
TL;DR-Box: 3–5 Bullet Points
H2: Was ist [Thema]?
H2: Für wen ist [Thema] sinnvoll?
H2: Wie funktioniert [Thema]? (Prozess, Schritte)
H2: Kosten und Aufwand (immer!)
H2: [Thema] bei [Unternehmensname]: Was wir anders machen
H2: Häufige Fragen (FAQ, min. 5 Fragen)
CTA: Jetzt anfragen / Termin buchen
Interne Links: Zu allen Cluster Pages ("Mehr über [Subtopic] →")
```

### Schritt 4: Cluster Pages schreiben (eine nach der anderen)

Jede Cluster Page:
- Rankt für ein spezifisches Long-Tail Keyword
- Geht tiefer als die Pillar Page beim Subtopic
- Hat eigenständigen Mehrwert (nicht einfach ausgebauter Pillar-Abschnitt)
- Verlinkt zurück zur Pillar Page (always)
- Verlinkt zu 1–2 thematisch verwandten Cluster Pages (wenn logisch)

Reihenfolge: Cluster Pages mit höchstem Suchvolumen oder direktem
Conversion-Bezug zuerst.

### Schritt 5: Pillar Page aktualisieren

Nach jeder neuen Cluster Page: Pillar Page updaten mit internem Link auf die neue Seite.
Auch "dateModified" im Schema updaten — Google sieht frische Inhalte.

### Schritt 6: Cluster für fertig erklären wenn:

- Pillar Page live und intern verlinkt
- Alle geplanten Cluster Pages live
- Jede Seite in Google Search Console indiziert
- Keine offensichtlichen Keyword-Kannibalisierungen zwischen den Pages
- Interne Links in beide Richtungen vorhanden

## Häufige Fehler beim Cluster-Aufbau

| Fehler | Konsequenz | Fix |
|---|---|---|
| Cluster Pages haben zu ähnlichen Themen | Keyword-Kannibalisierung | Klare Abgrenzung, Canonical prüfen |
| Pillar Page zu dünn (<800 Wörter) | Rankt nicht für breites Keyword | Ausbauen, Tiefe zeigen |
| Keine internen Links | Google sieht keinen Zusammenhang | Jede Seite verlinkt Pillar + zurück |
| Mehrere Cluster gleichzeitig halbfertig | Authority bleibt schwach | Einer nach dem anderen |
| Cluster Pages ohne eigene CTAs | Kein Conversion-Pfad | Jede Seite braucht eigenen CTA |
