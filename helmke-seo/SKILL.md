## 1. DAS FRAMEWORK — 3 PHASEN

### PHASE 1: MONEY PAGES FIRST

Bevor irgendetwas anderes passiert: **identifiziere und optimiere die 3–5 Seiten die tatsächlich Geld bringen.**

Money Pages = Seiten mit kommerziellem Intent wo Besucher zu Kunden werden:
- Leistungsseiten (nicht die Übersicht, sondern einzelne Leistungen)
- Standort-Landingpages (z.B. "IT Beratung Düsseldorf")
- Produkt/Angebot-Seiten (z.B. "KI Telefonassistent ab 59€/Monat")
- Kontakt / Anfrage-Seiten

**Jede Money Page braucht vor dem Cluster-Aufbau:**
1. ✅ Primär-Keyword in H1, Title, URL
2. ✅ Conversion-optimierter CTA above the fold
3. ✅ Vertrauenssignale (Kundenstimmen, Zahlen, Zertifikate, Logos)
4. ✅ LocalBusiness Schema.org
5. ✅ Meta Title (max 60 Zeichen) + Meta Description (max 155 Zeichen)
6. ✅ Interne Verlinkung von 3+ anderen Seiten auf diese Seite
7. ✅ Core Web Vitals: LCP < 2.5s, CLS < 0.1
8. ✅ FAQ-Section mit FAQPage Schema

→ Details: `references/money-pages.md`

---

### PHASE 2: EIN CLUSTER AUSWÄHLEN UND VOLLSTÄNDIG AUSBAUEN

Nachdem Money Pages stehen: Wähle **einen** Topic Cluster und baue ihn komplett aus — bevor du den nächsten anfängst.

**Cluster-Anatomie:**
```
Pillar Page (1x)      → Breites Hauptthema, 1.500–3.000 Wörter
  └── Cluster Page 1  → Spezifisches Subtopic, 800–1.500 Wörter
  └── Cluster Page 2  → Spezifisches Subtopic
  └── Cluster Page 3  → Spezifisches Subtopic
  └── ... (6–12 Cluster Pages pro Pillar)
```

**Regeln:**
- Alle Cluster Pages verlinken zurück zur Pillar Page
- Pillar Page verlinkt zu allen Cluster Pages
- Kein Cluster Page Thema überschneidet sich mit einer anderen
- Jede Seite rankt für exakt ein Haupt-Keyword

→ Details: `references/cluster-playbook.md`

---

### PHASE 3: WIEDERHOLEN — CLUSTER FÜR CLUSTER

Wenn Cluster 1 vollständig ist (alle Seiten live, intern verlinkt, indiziert):
→ Nächsten Cluster auswählen
→ Dieselbe Methodik anwenden
→ Nie mehrere Cluster gleichzeitig halbfertig aufbauen

**Priorität der Cluster nach Traffic-Potenzial und Business-Impact:**
1. Umsatz-nahe Themen (direkter Lead-Intent)
2. Vergleichsthemen (Käufer die evaluieren)
3. Problem-Awareness Themen (höheres Volumen, mehr Aufwand)
4. Brand-/Authority-Themen (E-E-A-T aufbauen)

## 2. UNIVERSAL CONTENT-REGELN (für alle Projekte)

### Seitenstruktur — immer einhalten

```
URL:          /[haupt-keyword-kebab-case]/
H1:           [Haupt-Keyword] + [Differenzierung] (einmal pro Seite)
H2:           Als Fragen formuliert ("Wie funktioniert...?", "Was kostet...?")
H3:           Spezifische Aspekte unter den H2s
Intro:        Haupt-Frage in ersten 200 Wörtern beantworten (GEO-Pflicht)
TL;DR-Box:    3–5 Bullet Points oben auf der Seite (AI-Extraktion)
FAQ:          Mindestens 5 echte Patientenfragen / Kundenfragen
CTA:          Immer vorhanden, above-the-fold und am Ende
```

### Copy-Regeln — was IMMER gilt

```
✅ Zahlen statt Adjektive: "800 Implantate/Jahr" statt "viele Implantate"
✅ Konkrete Differenzierung: Was macht DIESER Anbieter anders?
✅ Lokaler Kontext bei lokalen Seiten: echte Ortsreferenzen, Fahrzeit, Landmarks
✅ Author-Signal: Wer steht hinter diesem Content? (Name, Credentials)
✅ Aktualitäts-Signal: "Stand: [Monat Jahr]" bei informationellen Seiten
✅ Quellen: Externe Links zu autoritären Quellen (2–3 pro Seite)

❌ KI-Slop: "modernste Technologien", "höchste Qualität", "maßgeschneidert" ohne Inhalt
❌ Duplicate Content: Niemals >30% identischer Text zwischen zwei Seiten
❌ Keyword Stuffing: Haupt-Keyword max. 2% Dichte, natürlich fließend
❌ Thin Content: Unter 500 Wörter auf Leistungsseiten
❌ Leere Testimonials: "Thomas M., Zufriedener Kunde" — immer mit Kontext
```

### Title-Tag Formel

```
[Leistung] [Stadt/Region] | [konkreter USP] | [Markenname]
Beispiel: "IT Beratung Düsseldorf | KI & Automatisierung | Helmke Digital"
Beispiel: "Zahnarzt Krefeld | 800+ Implantate/Jahr | Kredent"
```

Max. 60 Zeichen. Keyword so früh wie möglich.

### Meta-Description Formel

```
[Problem des Besuchers] → [Lösung] + [USP]. [CTA mit Ort].
Beispiel: "IT Prozesse bremsen Ihr Unternehmen? Helmke Digital automatisiert mit KI und n8n — messbar, pragmatisch. Kostenlose Erstberatung in Düsseldorf."
```

Max. 155 Zeichen. Immer mit Action-Verb am Ende.

## 3. SCHEMA.ORG — PFLICHT auf jeder Seite

### Lokale Business-Seite (Minimum)
```json
{
  "@context": "https://schema.org",
  "@type": ["LocalBusiness", "ProfessionalService"],
  "name": "[Unternehmensname]",
  "url": "[URL]",
  "telephone": "[Telefon]",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "[Straße]",
    "addressLocality": "[Stadt]",
    "postalCode": "[PLZ]",
    "addressCountry": "DE"
  },
  "areaServed": { "@type": "State", "name": "Nordrhein-Westfalen" }
}
```

### FAQ-Section (Pflicht bei 3+ Fragen)
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "[Frage]",
    "acceptedAnswer": { "@type": "Answer", "text": "[Antwort]" }
  }]
}
```

### Artikel / Blog Post
```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "[Titel]",
  "datePublished": "[ISO-Datum]",
  "dateModified": "[ISO-Datum]",
  "author": { "@type": "Person", "name": "[Autorenname]" },
  "publisher": { "@type": "Organization", "name": "[Unternehmensname]" }
}
```

## 4. INTERNAL LINKING REGELN

- Jede neue Seite bekommt Links von mindestens 3 bestehenden Seiten
- Money Pages bekommen Links von 5+ Seiten
- Anker-Text: keyword-reich, nie "hier klicken" oder "mehr erfahren"
- Pillar Pages verlinken zu allen Cluster Pages
- Cluster Pages verlinken zur Pillar Page und zu 1–2 thematisch verwandten Cluster Pages
- Keine verwaisten Seiten (Seiten ohne eingehende interne Links)

## 5. 2026 SPEZIFIKA — GEO & AI OVERVIEWS

Google AI Overviews sind seit März 2025 in Deutschland aktiv.
Inhalte müssen für AI-Extraktion optimiert sein — nicht nur für Klicks.

**CITE-Struktur** (nach jeder H2 anwenden):
- **C**lear Answer Block: 40–60 Wörter, beantwortet die H2-Frage direkt
- **I**nclude Primärquellen: Jede Zahl mit Quellenlink
- **T**imestamp: "Stand Juni 2026" bei zeitkritischen Inhalten
- **E**ntity Authority: Echte Namen, echte Orte, echte Zahlen

**robots.txt** — NIEMALS blockieren:
- GPTBot
- ClaudeBot
- PerplexityBot
- Googlebot-Extended (AI Overviews)

→ Details: `references/geo-aeo.md`

## 6. CHECKLISTE VOR JEDER SEITE LIVE

```
□ H1 enthält Primär-Keyword (genau einmal)
□ Title Tag max. 60 Zeichen, Keyword am Anfang
□ Meta Description max. 155 Zeichen, endet mit CTA
□ URL: kurz, keyword-reich, kein Datum, kein ID
□ Canonical-Tag korrekt gesetzt (www. oder non-www — einheitlich)
□ Schema.org vorhanden (LocalBusiness + FAQPage wenn FA vorhanden)
□ Min. 3 interne Links auf diese Seite
□ Min. 2 externe Links zu Autoritätsquellen
□ Bilder: Alt-Text mit Keywords, WebP-Format, unter 200KB
□ Mobile: responsive, kein horizontales Scrollen
□ Core Web Vitals: LCP < 2.5s geprüft
□ Keine duplizierten Inhalte vs. andere Seiten (>30% identisch = Problem)
□ FAQ-Section vorhanden (min. 3 echte Fragen)
□ robots.txt: AI-Crawler nicht geblockt
```
